ADTF Filter Entwicklungsprozess

Veröffentlicht am 2018-08-28 | in ADTF

Automotive data and time-triggered framework (ADTF) ist die weltweit am häufigsten eingesetzte Entwicklungs- und Testumgebung für Fahrerassistenzsysteme. ADTF ist auch ein flexibles Framework. Man kann sein eignes Softwaremodul (auch als Filter genannt) je nach dem Bedarf programmieren. Als ich im Jahr 2016 bei Audi für mein Praktikum war, habe ich ADTF für mein Projekt benutzt. Damals habe ich meinen eigner Filter in ADTF programmiert, der die Frontkameraerkennung auf Basis von Mobileeye auswertet. Heute brauche ich immer noch mit ADTF am HIL zu arbeiten. Deswegen würde ich gerne das ADTF Filter Entwicklungsprozess zusammenfassen.

In ADTF werden viele Filter-Vorlage Dateien angeboten, die aus relevante C++ Dateien und Cmake Dateien bestehen. Ich würde hier den Empty-Filter als Beispiel nehmen. Für den Empty-Filter existiert so ein Ordner, der die folgende Dateien beinhaltet:

Tatsächlich wird ein Filter durch eine C++ Klasse realisiert. Die relevante C++ Klasse Dateien sind wie oben gezeigt. Zur Implementierung der C++ Klasse brauchen wir zwei Tools nämmlich Cmake und Visual Studio. CMake (cross-platform make) ist ein plattformunabhängiges Programmierwerkzeug für die Entwicklung und Erstellung von Software. Mit CMake werden aus Skriptdateien (CMakeLists.txt) Makefiles und Projekte für viele integrierte Entwicklungsumgebungen und Compiler erzeugt. Wir lassen Cmake die Solution Datei erzeugen, die im Visual Studio benutzt wird. Danach können wir die Solution Datei mit Visual Studio öffnen und je nach der Anforderungen weiter programmieren. Anschließend bilden wir die Source code und ein Plugin wird erzeugt. Am Ende integrieren wir das Plugin ins ADTF, indem wir im ADTF GUI die Verzeichnis des Plugins hinzufügen.

Der Prozess ist jetzt klar.

Schritt 1: CMake 2.8.4 und Visual Studio 2010 installieren

ADTF erfordert mindestens Cmake 2.8.4. Wenn man ADTF schon installiert hat, findet man auch die Installationsdatei von Cmake 2.8.4 unter ADTF Installationsverzeichnis wie “C:\Program Files\ADTF\2.13.0\bin\extern”.(Bei mir habe ich ADTF 2.13.0 installiert) Dann kann man die .exe Datei einfach doppelt klicken und Cmake installieren.

Bei Cmake 2.8.4 wird VS 2010 oft gewählt. Die Installation von VS 2010 erzähle ich hier nicht mehr.

Schritt 2: Solution Datei mit CMake erzeugen

Cmake GUI aufmachen. Es gibt Zwei Verzeichnisse, die wir manuell eingeben müssen. Zum Einen ist die Verzeichnis von source code, wo die obengenannte C++ und CMakeLists.txt liegen. Zum anderen ist die Verzeichnis “where to build the binaries”, wo die erzeugte Solution Datei abgelegt werden soll.

Hier muss man die Datei “CMakeLists.txt” beachten, die richtig je nach dem Bedarf implementiert werden muss. Bei mir habe ich so gemacht:

if(WIN32)
   cmake_minimum_required(VERSION 2.8.4 FATAL_ERROR)
elseif(WIN32)
   cmake_minimum_required(VERSION 2.6.0 FATAL_ERROR)
endif(WIN32)

cmake_policy(SET CMP0002 NEW)
cmake_policy(SET CMP0003 NEW)
cmake_policy(SET CMP0011 NEW)

project(adtf_examples)

find_package(ADTF REQUIRED)

if(SDL_LIBRARY)
if(WIN32)
    # because FindSDL.cmake is different between the various cmake versions
    # and because there is no variable which points to the dynamic library
    # this list handling is needed

    # get the first element of the possible list 
    # (SDLmain.lib and SDL.lib could be setted within SDL_LIBRARY)
    list(GET SDL_LIBRARY 0 FIRST_SDL_LIB)
    # replace the prefix lib with dll
    STRING(REPLACE ".lib" ".dll" SDL_SHARED_LIB_TEMP "${FIRST_SDL_LIB}")
    # check if the "new" library exists
    if(NOT EXISTS "${SDL_SHARED_LIB_TEMP}")
        # lib does not exist. Maybe this was the SDLmain.lib
        # get the length of the list
        list(LENGTH SDL_LIBRARY SDL_LIB_COUNT)
        # check if there is an other list element
        if(SDL_LIB_COUNT GREATER 1)
            # an other list element found
            list(GET SDL_LIBRARY 1 SECOND_SDL_LIB)
            # replace the prefix lib with dll
            STRING(REPLACE ".lib" ".dll" SDL_SHARED_LIB_TEMP "${SECOND_SDL_LIB}")
            # if this lib does also not exists, fall take the fallback of the cached user setted variable)
        endif(SDL_LIB_COUNT GREATER 1)
    endif(NOT EXISTS "${SDL_SHARED_LIB_TEMP}")
    # create a cached variable to make it changeable
    set(SDL_SHARED_LIB ${SDL_SHARED_LIB_TEMP} CACHE PATH "Path to the SDL shared lib. If path is not given or the file is not found, you have to copy the shared SDL library into the example/bin</debug> directory.")
    # check if the given dll exists
    if(EXISTS "${SDL_SHARED_LIB}")
        set(SDL_SHARED_LIB_FOUND TRUE CACHE INTERNAL "")
    else(EXISTS "${SDL_SHARED_LIB}")
        set(SDL_SHARED_LIB_FOUND FALSE CACHE INTERNAL "")
    endif(EXISTS "${SDL_SHARED_LIB}")
endif(WIN32)
endif(SDL_LIBRARY)

Cofigure starten. Es wird das Generator für das Projekt konfiguriert:

Finish klicken. Es wird ein Fehler angezeigt, der heißt “Error in configuration process, project files may be invalid”. Es ist normal. OK klicken.

Und zwei neue Verzeichnisse, die rot markiert werden, müssen wir eingeben. Zum Einen ist es “ADTF_DIR”, wobei wir die ADTF Installationsverzeichnis eingeben müssen. Zum anderen ist es “CMAKE_INSTALL_PREFIX”, wobei wir irgendwo eingeben können, damit das am Ende erzeugtes Plugin hierbei abgespeichert werden kann.

Cofigure wieder starten. Es würde viele neue in rot markierte Informationen geben. Keine Angst. Rot bedeutet nicht Fehler sodern nur neue Informationen. Unterns zeigt es “Cofiguring done”. Das bedeutet, dass Cofigure erfolgt.

Generate starten. Unterns zeigt es “Generating done”. Das bedeutet, dass Generate erfolgt.

Schritt 3: Solution Datei mit VS2010 aufmachen und programmieren

Solution Datei aufmachen. Dann kann man je nach dem Bedarf programmieren. Am Ende soll “Build Solution” ausgeführt werden. Dann bekommen wir das Plugin, das unter “CMAKE_INSTALL_PREFIX” abgespeichert wird. Das Plugin hat Dateityp “.plb”.

Schritt 4: Plugin ins ADTF integrieren

ADTF GUI aufmachen. Unter “Options\Settings\Globals\Global Plugin Directories” geben wir die Verzeichnis von Plugin ein. Dann können wir unseren eingener Filter in der Component Tree finden.
